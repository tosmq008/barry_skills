# 功能规格说明 - 待办事项管理系统

## 1. 核心功能规格

### F01 - 用户注册

**功能概述:**
- 功能目标: 新用户创建账号
- 目标用户: 未注册用户
- 使用场景: 首次使用系统

**功能入口:**
- 入口位置: 登录页底部
- 入口形式: "还没有账号？注册"链接

**功能流程:**
1. 用户点击注册链接
2. 跳转到注册页
3. 输入用户名、邮箱、密码
4. 系统实时校验输入
5. 点击注册按钮
6. 后端验证并创建账号
7. 自动登录并跳转首页

**业务规则:**
| 规则ID | 规则描述 | 触发条件 | 处理逻辑 |
|--------|----------|----------|----------|
| R01-01 | 邮箱唯一性 | 注册时 | 检查邮箱是否已存在 |
| R01-02 | 密码强度 | 输入密码时 | 至少8位，包含字母和数字 |
| R01-03 | 用户名长度 | 输入用户名时 | 2-20个字符 |

**数据要求:**
| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|
| username | string | ✅ | 2-20字符 | 用户名 |
| email | string | ✅ | 邮箱格式 | 邮箱地址 |
| password | string | ✅ | 8+字符，字母+数字 | 密码 |

**验收标准:**
- [ ] 邮箱格式校验正确
- [ ] 密码强度校验正确
- [ ] 邮箱重复时提示错误
- [ ] 注册成功后自动登录
- [ ] 注册失败时显示明确错误信息

---

### F02 - 用户登录

**功能概述:**
- 功能目标: 已注册用户登录系统
- 目标用户: 已注册用户
- 使用场景: 每次使用系统前

**功能流程:**
1. 用户输入邮箱和密码
2. 点击登录按钮
3. 后端验证凭据
4. 生成JWT Token
5. 返回Token和用户信息
6. 前端存储Token
7. 跳转首页

**业务规则:**
| 规则ID | 规则描述 | 触发条件 | 处理逻辑 |
|--------|----------|----------|----------|
| R02-01 | Token有效期 | 登录成功 | Token 7天有效 |
| R02-02 | 登录失败限制 | 连续失败5次 | 锁定10分钟 |

**验收标准:**
- [ ] 邮箱密码正确时登录成功
- [ ] 邮箱或密码错误时提示明确
- [ ] Token正确存储
- [ ] 登录后跳转首页

---

### F03 - 创建任务

**功能概述:**
- 功能目标: 快速创建新任务
- 目标用户: 已登录用户
- 使用场景: 需要记录新任务时

**功能入口:**
- 入口位置: 页面右下角悬浮按钮
- 入口形式: "+" 按钮

**功能流程:**
1. 用户点击"+"按钮
2. 弹出快速输入框
3. 输入任务标题
4. 按Enter或点击"添加"
5. 任务创建成功
6. 任务出现在列表顶部

**业务规则:**
| 规则ID | 规则描述 | 触发条件 | 处理逻辑 |
|--------|----------|----------|----------|
| R03-01 | 标题必填 | 创建任务 | 标题不能为空 |
| R03-02 | 默认状态 | 创建任务 | 状态为"待办" |
| R03-03 | 默认列表 | 创建任务 | 添加到当前列表 |
| R03-04 | 任务排序 | 创建任务 | 新任务在列表顶部 |

**数据要求:**
| 字段 | 类型 | 必填 | 校验规则 | 说明 |
|------|------|------|----------|------|
| title | string | ✅ | 1-200字符 | 任务标题 |
| description | text | ❌ | 0-2000字符 | 任务描述 |
| status | enum | ✅ | todo/in_progress/completed | 任务状态 |
| priority | enum | ❌ | high/medium/low/none | 优先级 |
| due_date | date | ❌ | 有效日期 | 截止日期 |
| list_id | int | ❌ | 存在的列表ID | 所属列表 |

**验收标准:**
- [ ] 标题为空时无法创建
- [ ] 按Enter快速创建
- [ ] 创建成功后输入框清空
- [ ] 新任务出现在列表顶部
- [ ] 创建失败时显示错误提示

### F04 - 标记任务完成

**功能概述:**
- 功能目标: 标记任务为已完成
- 目标用户: 已登录用户
- 使用场景: 完成任务后

**功能入口:**
- 入口位置: 任务项左侧
- 入口形式: 复选框

**功能流程:**
1. 用户点击任务复选框
2. 前端乐观更新（立即显示完成状态）
3. 发送API请求
4. 后端更新状态和完成时间
5. 返回更新后的任务
6. 前端确认更新成功
7. 播放完成动画

**业务规则:**
| 规则ID | 规则描述 | 触发条件 | 处理逻辑 |
|--------|----------|----------|----------|
| R04-01 | 记录完成时间 | 标记完成 | 设置completed_at为当前时间 |
| R04-02 | 状态变更 | 标记完成 | 状态改为"completed" |
| R04-03 | 可撤销 | 再次点击 | 取消完成状态 |

**验收标准:**
- [ ] 点击复选框立即显示完成状态
- [ ] 任务标题显示删除线
- [ ] 播放完成动画
- [ ] 再次点击可取消完成
- [ ] 网络失败时回滚状态

### F06 - 删除任务

**功能概述:**
- 功能目标: 删除不需要的任务
- 目标用户: 已登录用户

**功能入口:**

**业务规则:**
| 规则ID | 规则描述 | 触发条件 | 处理逻辑 |
|--------|----------|----------|----------|
| R06-01 | 二次确认 | 点击删除 | 弹出确认对话框 |

**验收标准:**
- [ ] 点击删除弹出确认对话框

### F07 - 搜索任务

**功能概述:**
- 功能目标: 快速找到任务
- 目标用户: 已登录用户
- 使用场景: 任务较多时快速定位

**功能入口:**
- 入口位置: 顶部栏
- 入口形式: 搜索框

**功能流程:**
1. 用户在搜索框输入关键词
2. 防抖300ms后发送请求
3. 后端搜索匹配任务
4. 返回搜索结果
5. 显示结果列表
6. 高亮匹配关键词

**搜索范围:**
- 任务标题
- 任务描述

**业务规则:**
| 规则ID | 规则描述 | 触发条件 | 处理逻辑 |
|--------|----------|----------|----------|
| R07-01 | 最小长度 | 输入搜索词 | 至少2个字符 |
| R07-02 | 防抖 | 输入搜索词 | 300ms防抖 |
| R07-03 | 模糊匹配 | 搜索 | 支持部分匹配 |

**验收标准:**
- [ ] 输入2个字符后开始搜索
- [ ] 搜索结果高亮关键词
- [ ] 无结果时显示空状态
- [ ] 清空搜索框恢复列表

## 2. API接口规格

### 2.1 认证接口

**POST /api/auth/register**
```json
Request:
{
  "username": "string",
  "email": "string",
  "password": "string"
}

Response:
{
  "user": {
    "id": 1,
    "username": "string",
    "email": "string"
  },
  "token": "string"
}
```

**POST /api/auth/login**
```json
Request:
{
  "email": "string",
  "password": "string"
}

Response:
{
  "user": {
    "id": 1,
    "username": "string",
    "email": "string"
  },
  "token": "string"
}
```

### 2.2 任务接口

**POST /api/tasks**
```json
Request:
{
  "title": "string",
  "description": "string",
  "status": "todo",
  "priority": "medium",
  "due_date": "2026-02-10",
  "list_id": 1
}

Response:
{
  "id": 1,
  "title": "string",
  "description": "string",
  "status": "todo",
  "priority": "medium",
  "due_date": "2026-02-10",
  "list_id": 1,
  "created_at": "2026-02-06T10:00:00Z",
  "updated_at": "2026-02-06T10:00:00Z"
}
```

**GET /api/tasks**
```json
Query Parameters:
- status: string (optional)
- priority: string (optional)
- list_id: int (optional)
- tag_id: int (optional)
- q: string (optional, search keyword)
- page: int (default: 1)
- limit: int (default: 50)

Response:
{
  "items": [
    {
      "id": 1,
      "title": "string",
      ...
    }
  ],
  "total": 100,
  "page": 1,
  "limit": 50
}
```
