# 待办事项应用 (Todo App) - PRD

## 1. 产品定位
一个简洁的待办事项管理 Web 应用，帮助用户管理日常任务。

## 2. 目标用户
- 个人用户
- 需要简单任务管理的人

## 3. 用户角色
| 角色 | 权限 |
|------|------|
| 普通用户 | 创建、查看、编辑、删除、完成待办事项 |

## 4. 核心功能

### 4.1 用户故事

#### US-01: 添加待办事项
**作为**用户，**我想要**添加一个新的待办事项，**以便**记录要完成的任务。

**验收标准：**
- 输入框在页面顶部
- 按回车键或点击添加按钮可添加
- 待办内容不能为空
- 待办内容最大长度 200 字符
- 添加成功后清空输入框
- 新待办自动出现在列表顶部

#### US-02: 查看待办列表
**作为**用户，**我想要**查看所有待办事项，**以便**了解当前任务。

**验收标准：**
- 列表按创建时间倒序展示
- 每项显示：内容、创建时间、完成状态
- 空列表显示"暂无待办事项"
- 支持筛选：全部/未完成/已完成

#### US-03: 完成待办事项
**作为**用户，**我想要**标记待办为已完成，**以便**跟踪任务进度。

**验收标准：**
- 点击复选框可标记完成/取消完成
- 已完成事项显示删除线样式
- 任务计数更新（如"3/5 已完成"）

#### US-04: 删除待办事项
**作为**用户，**我想要**删除待办项，**以便**清理不需要的任务。

**验收标准：**
- 鼠标悬停时显示删除按钮
- 点击删除后立即移除（无确认弹窗）
- 删除后计数更新

#### US-05: 编辑待办事项
**作为**用户，**我想要**修改已有待办内容，**以便**更改任务描述。

**验收标准：**
- 双击待办内容进入编辑模式
- 编辑完按回车保存，按 Esc 取消
- 编辑后内容不能为空
- 编辑后内容最大长度 200 字符

## 5. 技术要求
- 前端：HTML + CSS + JavaScript（纯前端，无后端）
- 数据存储：localStorage
- 响应式设计

## 6. 非功能需求
- NFR-01: 页面加载时间 < 1 秒
- NFR-02: 数据持久化到 localStorage
- NFR-03: 支持 Chrome、Firefox、Safari 浏览器
