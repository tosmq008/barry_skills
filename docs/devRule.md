
# 开发任务拆分必须要遵守以下规则
## 后端根据业务规则变化进行评估设计，建立后端业务变更任务
## 根据业务业务规则变化，建立数据库变更任务
## 根据数据库变更，建立数据库变更任务
## 评估越设计API的变化，建立API变更任务
### 根据API设计的变化，更新或产出API文档
### 根据API变更，建立后端API变更任务
### 根据API变更，建立前端代码改动逻辑
## 设计与评估前端界面的变化，建立UI变更任务
## 创建前后端联调任务
## 执行修改后的冒烟测试
### 建立测试用例清单
### 冒烟测试不通过，则修改代码 和 文档
### 直到冒烟测试通过
