# PRD 输出模板

## 概述

本文档定义产品需求文档(PRD)的完整输出模板，确保PRD文档完整、清晰、可执行。

## 三、PRD编写规范

### 3.1 文档规范

| 规范项 | 要求 |
|--------|------|
| 语言 | 清晰、准确、无歧义 |
| 格式 | 统一使用Markdown |
| 命名 | 文件名使用小写+连字符 |
| 版本 | 每次修改更新版本号 |
| 状态 | 标注文档状态 |

### 3.2 内容规范

| 规范项 | 要求 |
|--------|------|
| 完整性 | 覆盖所有功能和状态 |
| 一致性 | 术语和格式统一 |
| 可执行 | 开发可直接依据实现 |
| 可验收 | 有明确验收标准 |
| 可追溯 | 需求可追溯到目标 |
