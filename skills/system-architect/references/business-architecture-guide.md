# 业务架构设计指南

## 1. 领域驱动设计（DDD）

### 1.1 战略设计

**限界上下文划分原则：**
1. 语言边界 - 同一术语在不同上下文含义不同时拆分
2. 团队边界 - 不同团队负责不同上下文
3. 业务边界 - 独立的业务流程形成独立上下文
4. 数据边界 - 数据模型差异大时拆分

**上下文映射模式：**

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| 共享内核 | 两个上下文共享部分模型 | 紧密协作的团队 |
| 客户-供应商 | 上游提供，下游消费 | 明确的依赖关系 |
| 防腐层 | 隔离外部模型影响 | 集成第三方/遗留系统 |
| 开放主机服务 | 提供标准化协议 | 多消费方场景 |
| 发布语言 | 共享的数据交换格式 | 跨上下文数据交换 |
| 各行其道 | 无集成关系 | 完全独立的上下文 |

### 1.2 战术设计

**聚合设计原则：**
1. 聚合内保持事务一致性
2. 聚合间通过领域事件实现最终一致性
3. 聚合根是外部访问的唯一入口
4. 聚合尽量小，只包含必要的实体和值对象

**领域对象分类：**

| 对象类型 | 特征 | 示例 |
|----------|------|------|
| 实体 | 有唯一标识，可变 | User, Order |
| 值对象 | 无标识，不可变 | Address, Money |
| 聚合根 | 实体的根，事务边界 | Order (包含OrderItem) |
| 领域服务 | 无状态的业务逻辑 | PricingService |
| 领域事件 | 业务状态变化通知 | OrderCreated |
| 仓储 | 聚合的持久化接口 | OrderRepository |

## 2. 业务流程建模

### 2.1 流程建模方法

**BPMN 核心元素：**
- 事件（开始/中间/结束）
- 活动（任务/子流程）
- 网关（排他/并行/包容）
- 连接（顺序流/消息流）

### 2.2 业务流程分析模板

```
流程名称: [名称]
流程目标: [目标]
触发条件: [条件]
参与角色: [角色列表]

主流程:
  1. [角色A] → [动作1] → [输出1]
  2. [角色B] → [动作2] → [输出2]
  3. [系统] → [自动处理] → [输出3]

异常流程:
  2a. [条件] → [异常处理] → [恢复/终止]

流程指标:
  - 平均处理时长: [时间]
  - 异常率: [百分比]
```

### 2.3 典型业务流程示例

**电商订单流程：**

```
用户下单 → 库存校验 → 订单创建 → 支付处理 → 支付回调
    │          │          │          │          │
    │       库存不足     生成订单号   调用支付   更新订单状态
    │       → 提示用户   → 锁定库存  → 跳转支付  → 通知发货
    │                                            │
    │                                     支付超时/失败
    │                                     → 释放库存
    │                                     → 关闭订单
    ▼
  订单完成 ← 确认收货 ← 物流配送 ← 仓库发货
```

**用户注册流程：**

```
输入信息 → 格式校验 → 唯一性检查 → 创建账户 → 发送验证 → 激活账户
    │          │          │          │          │          │
  手机/邮箱  前端校验   查询DB     写入DB    短信/邮件   验证链接
  密码/验证码 后端校验   已存在→提示 生成Token  异步发送   更新状态
```

## 3. 业务能力建模

### 3.1 能力地图构建步骤

1. 识别一级业务能力（战略层）
2. 分解二级业务能力（业务域层）
3. 细化三级业务能力（功能层）
4. 映射能力到系统组件
5. 识别能力间的依赖关系

### 3.2 能力成熟度评估

| 成熟度 | 说明 | 特征 |
|--------|------|------|
| L1 初始 | 手工/临时 | 无标准流程 |
| L2 可重复 | 有基本流程 | 依赖个人经验 |
| L3 已定义 | 标准化流程 | 组织级规范 |
| L4 已管理 | 量化管理 | 数据驱动决策 |
| L5 优化 | 持续改进 | 自动化+智能化 |

### 3.3 能力与系统映射

| 业务能力 | 系统组件 | 服务 | 数据实体 | 成熟度 |
|----------|----------|------|----------|--------|
| 用户管理 | 用户中心 | user-service | User, Role | L3 |
| 订单处理 | 交易中心 | order-service | Order, OrderItem | L2 |
| 内容管理 | 内容平台 | content-service | Article, Category | L2 |

## 4. 业务规则管理

### 4.1 业务规则分类

| 规则类型 | 说明 | 示例 | 实现方式 |
|----------|------|------|----------|
| 约束规则 | 数据必须满足的条件 | 年龄≥18岁 | 校验逻辑 |
| 计算规则 | 数据的计算方式 | 折扣=原价×折扣率 | 计算引擎 |
| 推断规则 | 基于条件的推断 | VIP用户享受免运费 | 规则引擎 |
| 流程规则 | 流程的控制逻辑 | 金额>1000需审批 | 工作流引擎 |
