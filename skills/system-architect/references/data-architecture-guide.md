# 数据架构设计指南

## 1. 数据建模方法

### 1.1 建模流程

```
业务需求 → 概念模型 → 逻辑模型 → 物理模型 → DDL脚本
```

### 1.2 概念模型设计

**实体识别方法：**
1. 名词分析法 - 从需求文档提取名词作为候选实体
2. 事件分析法 - 从业务事件反推相关实体
3. 角色分析法 - 从用户角色推导管理的实体

**关系类型：**
| 关系 | 说明 | 实现方式 |
|------|------|----------|
| 1:1 | 一对一 | 外键/同表 |
| 1:N | 一对多 | 外键 |
| M:N | 多对多 | 关联表 |
| 自关联 | 树形结构 | parent_id |

### 1.3 逻辑模型设计

**范式化设计：**
- 1NF: 字段原子性
- 2NF: 消除部分依赖
- 3NF: 消除传递依赖

**反范式化场景：**
| 场景 | 反范式策略 | 收益 |
|------|------------|------|
| 高频查询 | 冗余字段 | 减少JOIN |
| 统计报表 | 汇总表 | 加速查询 |
| 历史快照 | 快照字段 | 数据追溯 |

## 2. 数据存储架构

### 2.1 多模型存储策略

| 数据特征 | 存储引擎 | 选型依据 |
|----------|----------|----------|
| 结构化事务数据 | PostgreSQL/MySQL | ACID事务，复杂查询 |
| 半结构化文档 | MongoDB | 灵活Schema，嵌套数据 |
| 键值缓存 | Redis | 高性能读写，数据结构丰富 |
| 全文搜索 | Elasticsearch | 倒排索引，聚合分析 |
| 时序数据 | TimescaleDB/InfluxDB | 时间序列优化 |
| 图关系数据 | Neo4j | 复杂关系查询 |
| 文件/对象 | MinIO/OSS/S3 | 大文件存储 |

### 2.2 数据分层架构

```
┌─────────────────────────────────────────┐
│  ODS层 (原始数据层)                      │
│  原始业务数据，保持原貌                   │
├─────────────────────────────────────────┤
│  DWD层 (明细数据层)                      │
│  清洗、标准化后的明细数据                 │
├─────────────────────────────────────────┤
│  DWS层 (汇总数据层)                      │
│  按主题汇总的宽表数据                     │
├─────────────────────────────────────────┤
│  ADS层 (应用数据层)                      │
│  面向应用的指标数据                       │
└─────────────────────────────────────────┘
```

## 3. 数据流设计

### 3.1 实时数据流

```
数据源 → 消息队列(Kafka) → 流处理(Flink/Spark) → 存储/服务
```

### 3.2 批量数据流

```
数据源 → ETL工具(Airflow/DolphinScheduler) → 数据仓库 → BI报表
```

### 3.3 CDC（变更数据捕获）

| 方案 | 实现方式 | 延迟 | 侵入性 |
|------|----------|------|--------|
| 日志解析 | Debezium/Canal | 秒级 | 无侵入 |
| 触发器 | 数据库触发器 | 实时 | 有侵入 |
| 轮询 | 定时查询变更 | 分钟级 | 无侵入 |
| 双写 | 应用层双写 | 实时 | 有侵入 |

## 4. 数据一致性

### 4.1 分布式事务方案

| 方案 | 一致性 | 性能 | 复杂度 | 适用场景 |
|------|--------|------|--------|----------|
| 2PC | 强一致 | 低 | 中 | 数据库层面 |
| TCC | 强一致 | 中 | 高 | 业务层面 |
| Saga | 最终一致 | 高 | 中 | 长事务 |
| 本地消息表 | 最终一致 | 高 | 低 | 异步场景 |
| 事务消息 | 最终一致 | 高 | 低 | MQ支持场景 |

### 4.2 幂等性设计

| 场景 | 幂等方案 | 实现方式 |
|------|----------|----------|
| 创建操作 | 唯一键约束 | 数据库唯一索引 |
| 更新操作 | 乐观锁 | 版本号/时间戳 |
| 支付操作 | 幂等Token | Redis存储Token |
| 消息消费 | 消费记录表 | 消息ID去重 |

## 5. 数据治理

### 5.1 数据质量管理

| 质量维度 | 检查规则 | 处理方式 |
|----------|----------|----------|
| 完整性 | 必填字段非空 | 拒绝/默认值 |
| 准确性 | 格式/范围校验 | 拒绝/修正 |
| 一致性 | 跨表数据一致 | 对账/修复 |
| 时效性 | 数据更新及时 | 监控告警 |
| 唯一性 | 无重复数据 | 去重/合并 |

### 5.2 数据安全分级

| 级别 | 数据类型 | 保护措施 |
|------|----------|----------|
| 公开 | 公开信息 | 无特殊要求 |
| 内部 | 业务数据 | 访问控制 |
| 机密 | 用户隐私(手机/身份证) | 加密存储+脱敏展示 |
| 绝密 | 密钥/凭证 | KMS管理+审计 |
