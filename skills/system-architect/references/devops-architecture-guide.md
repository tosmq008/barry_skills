# 运维架构设计指南

## 1. 部署架构

### 1.1 容器化部署

**Docker 最佳实践：**
- 多阶段构建减小镜像体积
- 非root用户运行
- 健康检查配置
- 合理的层缓存策略

**Dockerfile 规范：**
```dockerfile
# 构建阶段
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production
COPY . .
RUN npm run build

# 运行阶段
FROM node:20-alpine
RUN addgroup -g 1001 appgroup && adduser -u 1001 -G appgroup -s /bin/sh -D appuser
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
USER appuser
HEALTHCHECK --interval=30s CMD wget -q --spider http://localhost:3000/health
EXPOSE 3000
CMD ["node", "dist/main.js"]
```

### 1.2 编排方案选型

| 方案 | 适用规模 | 复杂度 | 优势 |
|------|----------|--------|------|
| Docker Compose | 小型/开发 | 低 | 简单快速 |
| Docker Swarm | 中小型 | 中 | Docker原生 |
| Kubernetes | 中大型 | 高 | 功能完善，生态丰富 |
| Serverless | 事件驱动 | 低 | 免运维，按需付费 |

### 1.3 Kubernetes 部署架构

**核心资源配置：**

| 资源类型 | 用途 | 关键配置 |
|----------|------|----------|
| Deployment | 无状态服务 | replicas, resources, probes |
| StatefulSet | 有状态服务 | volumeClaimTemplates |
| Service | 服务发现 | ClusterIP/NodePort/LoadBalancer |
| Ingress | 外部访问 | 路由规则, TLS |
| ConfigMap | 配置管理 | 非敏感配置 |
| Secret | 密钥管理 | 敏感配置 |
| HPA | 自动扩缩 | CPU/内存/自定义指标 |

## 2. CI/CD 流水线

### 2.1 流水线设计

**标准流水线阶段：**

```
┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐
│ 代码 │→│ 检查 │→│ 测试 │→│ 构建 │→│ 部署 │→│ 验证 │
│ 提交 │  │ 阶段 │  │ 阶段 │  │ 阶段 │  │ 阶段 │  │ 阶段 │
└──────┘  └──────┘  └──────┘  └──────┘  └──────┘  └──────┘
            │          │          │          │          │
          Lint       单元测试   Docker    K8s/CD    健康检查
          类型检查   集成测试   Build     灰度发布   冒烟测试
          安全扫描   覆盖率    镜像推送   蓝绿部署   回归测试
```

### 2.2 CI/CD 工具选型

| 工具 | 类型 | 适用场景 | 特点 |
|------|------|----------|------|
| GitHub Actions | 云端CI | GitHub项目 | 集成度高，免费额度 |
| GitLab CI | 自建CI | GitLab项目 | 功能完善，自建可控 |
| Jenkins | 自建CI | 复杂流水线 | 插件丰富，灵活 |
| ArgoCD | GitOps CD | K8s部署 | 声明式，自动同步 |

### 2.3 发布策略

| 策略 | 说明 | 风险 | 回滚速度 |
|------|------|------|----------|
| 滚动更新 | 逐步替换实例 | 低 | 中 |
| 蓝绿部署 | 两套环境切换 | 低 | 快 |
| 金丝雀发布 | 小流量验证 | 最低 | 快 |
| A/B测试 | 按用户分流 | 低 | 快 |

### 3.2 关键监控指标

**RED 方法（服务级）：**
- Rate: 请求速率（QPS）
- Errors: 错误率
- Duration: 请求延迟

**USE 方法（资源级）：**
- Utilization: 利用率
- Saturation: 饱和度
- Errors: 错误数

### 3.3 告警规则设计

| 级别 | 响应时间 | 通知方式 | 示例 |
|------|----------|----------|------|
| P0 紧急 | 5分钟 | 电话+短信+IM | 服务不可用 |
| P1 严重 | 15分钟 | 短信+IM | 错误率>5% |
| P2 警告 | 30分钟 | IM | CPU>80% |
| P3 提示 | 4小时 | 邮件 | 磁盘>70% |

## 4. 高可用设计

### 4.1 容灾等级

| 等级 | 说明 | RTO | RPO | 成本 |
|------|------|-----|-----|------|
| 冷备 | 定期备份，手动恢复 | 小时级 | 小时级 | 低 |
| 温备 | 实时同步，手动切换 | 分钟级 | 分钟级 | 中 |
| 热备 | 实时同步，自动切换 | 秒级 | 秒级 | 高 |
| 多活 | 多地同时服务 | 0 | 0 | 极高 |

### 4.2 故障恢复流程

```
故障发现 → 告警通知 → 故障定位 → 应急处理 → 根因分析 → 改进措施
  │           │          │          │          │          │
 监控       值班人员    日志/链路   降级/切换   5Why分析   预防措施
 告警       响应确认    指标分析   回滚/扩容   复盘报告   自动化
```

### 4.3 备份策略

| 备份类型 | 频率 | 保留期限 | 存储位置 |
|----------|------|----------|----------|
| 全量备份 | 每日 | 30天 | 异地存储 |
| 增量备份 | 每小时 | 7天 | 本地+异地 |
| Binlog | 实时 | 7天 | 本地 |
| 配置备份 | 每次变更 | 永久 | Git仓库 |

## 5. 日志管理

### 5.1 日志规范

**日志级别使用：**
| 级别 | 使用场景 | 示例 |
|------|----------|------|
| ERROR | 需要立即处理的错误 | 数据库连接失败 |
| WARN | 潜在问题，不影响主流程 | 重试成功 |
| INFO | 关键业务流程节点 | 订单创建成功 |
| DEBUG | 调试信息 | 请求参数详情 |

**结构化日志格式：**
```json
{
  "timestamp": "2024-01-01T00:00:00Z",
  "level": "INFO",
  "service": "order-service",
  "trace_id": "xxx",
  "message": "Order created",
  "data": { "order_id": "123" }
}
```
