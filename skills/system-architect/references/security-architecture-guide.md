# 安全架构设计指南

## 1. 安全威胁建模

### 1.1 STRIDE 模型

| 威胁 | 全称 | 安全属性 | 防御方向 |
|------|------|----------|----------|
| S | Spoofing (仿冒) | 认证 | 身份验证机制 |
| T | Tampering (篡改) | 完整性 | 数据签名/校验 |
| R | Repudiation (抵赖) | 不可否认 | 审计日志 |
| I | Information Disclosure (泄露) | 保密性 | 加密/访问控制 |
| D | Denial of Service (拒绝服务) | 可用性 | 限流/冗余 |
| E | Elevation of Privilege (提权) | 授权 | 最小权限原则 |

### 1.2 OWASP Top 10 防御

| 风险 | 防御措施 | 实现方式 |
|------|----------|----------|
| 注入攻击 | 参数化查询 | ORM/预编译语句 |
| 认证失效 | 强认证机制 | MFA/密码策略 |
| 敏感数据泄露 | 加密保护 | TLS/AES/脱敏 |
| XXE | 禁用外部实体 | XML解析器配置 |
| 访问控制失效 | RBAC/ABAC | 中间件权限检查 |
| 安全配置错误 | 安全基线 | 自动化检查 |
| XSS | 输出编码 | CSP/框架转义 |
| 反序列化漏洞 | 输入验证 | 白名单校验 |
| 组件漏洞 | 依赖审计 | SCA工具扫描 |
| 日志监控不足 | 完善监控 | SIEM/告警 |

## 2. 认证安全

### 2.1 密码安全

| 措施 | 实现 | 说明 |
|------|------|------|
| 密码哈希 | bcrypt/argon2id | 不可逆存储 |
| 盐值 | 每用户独立盐值 | 防彩虹表 |
| 密码策略 | 长度≥8，复杂度要求 | 防弱密码 |
| 登录限制 | 5次失败锁定15分钟 | 防暴力破解 |
| 验证码 | 图形/滑块验证码 | 防自动化攻击 |

### 2.2 Token 安全

| 措施 | 实现 | 说明 |
|------|------|------|
| 签名算法 | RS256/ES256 | 非对称签名 |
| 有效期 | Access: 15min, Refresh: 7d | 短期+刷新 |
| 存储方式 | HttpOnly Cookie / 内存 | 防XSS窃取 |
| Token撤销 | 黑名单/版本号 | 强制下线 |
| 刷新机制 | Refresh Token Rotation | 防Token泄露 |

### 2.3 OAuth 2.0 / OIDC

**授权流程选择：**
| 流程 | 适用场景 | 安全级别 |
|------|----------|----------|
| Authorization Code + PKCE | SPA/移动端 | 高 |
| Authorization Code | 服务端Web | 高 |
| Client Credentials | 服务间调用 | 中 |

## 3. 数据安全

### 3.1 加密策略

| 场景 | 加密方式 | 算法 | 密钥管理 |
|------|----------|------|----------|
| 传输加密 | TLS 1.3 | ECDHE+AES-GCM | 证书管理 |
| 存储加密 | 字段级加密 | AES-256-GCM | KMS |
| 密码存储 | 哈希 | argon2id/bcrypt | 盐值 |
| 敏感配置 | 配置加密 | AES-256 | Vault/KMS |

### 3.2 数据脱敏

| 数据类型 | 脱敏规则 | 示例 |
|----------|----------|------|
| 手机号 | 中间4位 | 138****5678 |
| 身份证 | 中间8位 | 110***********1234 |
| 邮箱 | @前部分 | a***@example.com |
| 银行卡 | 保留后4位 | ************1234 |
| 姓名 | 保留姓 | 张** |

### 3.3 数据访问控制

**RBAC 模型：**
```
用户 → 角色 → 权限 → 资源
User → Role → Permission → Resource
```

**ABAC 模型（高级）：**
```
主体属性 + 资源属性 + 环境属性 + 操作 → 决策
Subject + Resource + Environment + Action → Decision
```

## 4. API 安全

### 4.1 输入验证

| 验证项 | 规则 | 实现 |
|--------|------|------|
| 类型检查 | 严格类型匹配 | Schema验证(Zod/Pydantic) |
| 长度限制 | 最大长度约束 | 字段级限制 |
| 格式校验 | 正则/枚举 | 白名单校验 |
| 文件校验 | 类型/大小/内容 | Magic Number检查 |

### 4.2 限流策略

| 策略 | 算法 | 适用场景 |
|------|------|----------|
| 固定窗口 | 计数器 | 简单限流 |
| 滑动窗口 | 滑动计数 | 平滑限流 |
| 令牌桶 | Token Bucket | 允许突发 |
| 漏桶 | Leaky Bucket | 恒定速率 |

### 4.3 请求安全

| 措施 | 实现 | 防御目标 |
|------|------|----------|
| 请求签名 | HMAC-SHA256 | 防篡改 |
| 时间戳 | ±5分钟有效 | 防重放 |
| Nonce | 唯一随机数 | 防重放 |
| IP白名单 | 服务间调用 | 防未授权访问 |

## 5. 安全运营

### 5.1 安全审计

| 审计项 | 记录内容 | 保留期限 |
|--------|----------|----------|
| 登录日志 | 时间/IP/设备/结果 | 180天 |
| 操作日志 | 用户/操作/对象/结果 | 365天 |
| 数据访问 | 查询/导出/修改 | 365天 |
| 系统变更 | 配置/权限/部署 | 永久 |

### 5.2 安全监控告警

| 监控项 | 告警条件 | 响应级别 |
|--------|----------|----------|
| 暴力破解 | 同IP 10次失败/分钟 | P1 |
| 异常登录 | 异地/异设备登录 | P2 |
| 敏感操作 | 批量数据导出 | P2 |
| 权限变更 | 管理员权限变更 | P1 |
| 漏洞告警 | CVE高危漏洞 | P0 |
