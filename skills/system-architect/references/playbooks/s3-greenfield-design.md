# S3: 新系统设计执行手册

> 重在“全面”——从零开始设计完整系统架构，所有 Phase 完整执行。

## 适用场景

- 全新项目从零开始
- 独立子系统的完整设计
- 产品重构（完全重写）

## 输入与输出

| 项目 | 内容 |
|------|------|
| **输入** | PRD / 产品方案（product-expert 输出） |
| **输出** | `docs/architecture/technical-architecture-design.md` |
| **交付对象** | tech-manager（流转） |

## 执行步骤

### Step 1: Phase 1 完整需求与系统分析

- 读取 PRD 全套文档
- 功能需求分解（功能分解矩阵）
- 非功能需求量化（NFR 评估矩阵）
- 系统边界与约束识别

### Step 2: Phase 2 业务功能设计与关联分析

- 梳理角色、场景、流程、状态、业务规则
- 输出功能关联矩阵和关键业务闭环
- 明确哪些能力必须同域设计，哪些能力应解耦

### Step 3: Phase 3 功能架构与模块抽象

- 基于 `L1-feature-architecture` 抽象核心模块、支撑模块、平台能力模块、外部集成模块
- 建立功能节点到抽象模块的映射关系
- 明确跨模块协作方式和边界保护方式

### Step 4: Phase 4 工程架构设计

- 将抽象模块落到服务、应用、包、目录、workspace 与共享库
- 明确工程模块边界、依赖方向、共享能力与扩展点
- 形成可供 tech-manager 直接拆任务的模块视图

### Step 5: Phase 5 技术架构设计与核心选型

- 技术选型评估（评估矩阵 + 必要时 web 搜索）
- 架构模式选择
- 运行链路、代码分层、扩展策略设计
- ADR 记录
- **遵循 `SKILL.md` 的 Rule 4：Phase 5 完成后暂停，待用户确认后再进入详细子架构**

### Step 6: Phase 6-11 详细子架构设计

- **Phase 6**：后端架构设计
- **Phase 7**：前端架构设计
- **Phase 8**：集成架构设计
- **Phase 9**：数据架构设计
- **Phase 10**：安全架构设计
- **Phase 11**：运维与交付架构设计

### Step 7: Phase 12 输出完整方案 → 流转 tech-manager

流转动作（详见 `references/handoff-guide.md`）：
1. 验证交付物完整性（主文档 + 系统分析 + 功能架构 + 工程架构 + API契约 + 数据字典 + ADR）
2. 在流转指令中包含完整架构方案、全量 API 清单和数据模型
3. 调用 tech-manager skill，传递流转指令
4. tech-manager 从零搭建项目，按实施计划分阶段推进

## S3 特有指引

### 技术选型 POC 验证流程

对于关键技术选型（框架、数据库、中间件），必须进行 POC 验证：

```text
1. 定义 POC 目标（验证什么？成功标准是什么？）
2. 搭建最小验证环境
3. 实现核心场景（2-3 个关键用例）
4. 收集数据（性能指标、开发体验、问题清单）
5. 输出 POC 报告 → 纳入 ADR 决策依据
```

POC 触发条件（满足任一）：
- 团队从未使用过该技术
- 候选方案评分差距 < 10%，难以决策
- 涉及性能敏感场景（高并发/大数据量）
- 涉及付费/商业许可的技术

### MVP 范围界定方法

| 步骤 | 动作 | 输出 |
|------|------|------|
| 1. 功能分级 | 按 MoSCoW 对功能分级 | Must/Should/Could/Won't |
| 2. MVP 圈定 | MVP = 全部 Must Have + 部分 Should Have | MVP 功能清单 |
| 3. 架构裁剪 | MVP 阶段可简化的工程/技术决策 | 简化方案（如先模块化单体） |
| 4. 预留接缝 | 标记 MVP 后需要扩展的架构点 | 扩展点清单 |

### 架构评审会议组织建议

```text
评审时机: Phase 5 技术架构设计完成后、Phase 12 最终输出前
参与角色: 架构师(主讲) + 技术负责人 + 前端/后端 Tech Lead + 运维
评审内容:
  - 功能架构与工程架构边界合理性
  - 技术选型合理性（ADR 逐项过）
  - 架构模式匹配度
  - 性能/安全/运维方案可行性
  - 实施计划风险评估
评审输出: 评审意见 → 方案修订 → 定稿
```

## Phase 执行范围

所有 Phase（1-12）**完整执行，无裁剪**。
