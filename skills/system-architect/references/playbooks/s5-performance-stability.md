# S5: 性能/稳定性专项执行手册

> 重在"定点"——针对性能瓶颈或稳定性问题，设计精准优化方案。

## 适用场景

- 系统响应变慢，用户体验下降
- 大促/活动前的性能保障
- SLA 不达标需要优化
- 频繁出现稳定性问题（OOM、超时、雪崩）
- 容量规划（业务增长前的扩容评估）

## 输入与输出

| 项目 | 内容 |
|------|------|
| **输入** | 性能数据 + 监控告警 + 现有架构 |
| **输出** | `docs/architecture/performance-optimization-plan.md` |
| **交付对象** | tech-manager（流转） |

## 执行步骤

### Step 1: 瓶颈识别与根因定位

数据收集：
- APM 数据：接口响应时间分布、慢接口 Top N
- 数据库：慢查询日志、连接池使用率、锁等待
- 缓存：命中率、内存使用、热 Key
- 系统资源：CPU/内存/磁盘IO/网络
- 前端：Core Web Vitals、资源加载瀑布图

瓶颈分类：

```
瓶颈类型          典型表现                    定位方法
──────────────────────────────────────────────────────────
计算瓶颈          CPU 持续高位               Profiling/火焰图
IO 瓶颈           磁盘IO高/网络延迟          iostat/网络抓包
内存瓶颈          OOM/频繁GC                 内存分析/GC日志
数据库瓶颈        慢查询/连接池满            慢查询日志/explain
缓存瓶颈          缓存穿透/击穿/雪崩        命中率监控
并发瓶颈          线程池满/锁竞争            线程dump/锁分析
前端瓶颈          FCP/LCP超标               Lighthouse/DevTools
```

### Step 3: 优化方案分级

| 级别 | 定义 | 示例 | 实施周期 |
|------|------|------|----------|
| Quick Win | 低成本高收益 | 加索引、调连接池、开缓存 | 1-3天 |
| 中期优化 | 需要一定开发量 | 异步化、读写分离、CDN | 1-2周 |
| 长期架构调整 | 架构级变更 | 分库分表、微服务拆分、CQRS | 1-3月 |

### Step 4: 压测验证计划

```
压测流程：
1. 基准测试（优化前）→ 记录基线数据
2. 实施 Quick Win 优化
3. 回归压测 → 对比基线
4. 实施中期优化
5. 回归压测 → 对比基线
6. 评估是否需要长期架构调整
```

### Step 5: SLA 目标定义

| 指标 | 当前值 | 优化目标 | 验收标准 |
|------|--------|----------|----------|
| P50 响应时间 | [ms] | [ms] | 压测验证 |
| P99 响应时间 | [ms] | [ms] | 压测验证 |
| 可用率 | [%] | [%] | 监控验证 |
| 错误率 | [%] | [%] | 监控验证 |

### Step 6: 输出优化方案 → 流转 tech-manager

流转动作（详见 `references/handoff-guide.md`）：
1. 验证交付物完整性（优化方案 + 压测验证计划 + SLA目标）
2. 在流转指令中标注优化分级（Quick Win / 中期 / 长期）和实施顺序
3. 调用 tech-manager skill，传递流转指令
4. tech-manager 按优先级分批实施，每批完成后需压测验证

## Phase 执行范围

| Phase | S5 执行模式 |
|-------|-------------|
| Phase 1 | 瓶颈识别与根因定位（Profiling/APM/慢查询） |
| Phase 2 | ○按需（仅当性能问题与业务域划分相关时） |
| Phase 3 | 技术架构优化方案 |
| Phase 4 | 后端性能优化：缓存、并发、数据库、连接池 |
| Phase 5 | 前端性能优化：加载、渲染、资源、缓存 |
| Phase 6 | ○按需（仅当前后端交互是性能瓶颈时） |
| Phase 7 | 数据层优化：索引、查询、分库分表、缓存同步 |
| Phase 8 | ○按需（确保性能优化不引入安全漏洞） |
| Phase 9 | 运维层优化：监控增强、扩缩容策略、资源调优 |
