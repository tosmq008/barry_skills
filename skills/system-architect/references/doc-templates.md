# 文档输出模板

> 各场景输出文档的模板结构。每完成一个 Phase，立即将该 Phase 的设计成果追加写入文档。

## 分批写入规则

1. **每完成一个 Phase，立即将该 Phase 的设计成果追加写入文档**
2. **每次写入控制在合理篇幅内，避免一次性大量输出**
3. **写入顺序严格按照 Phase 顺序执行**
4. **每次写入前先读取文档当前内容，确保追加位置正确**
5. **写入后验证文档结构完整性**
6. **S1/S5 场景跳过的 Phase 不写入对应章节**

## 输出文件路径（按场景）

| 场景 | 主文档路径 | 说明 |
|------|------------|------|
| S1 架构评审 | `docs/architecture/architecture-review-report.md` | 架构评审报告 |
| S2 增量需求 | `docs/architecture/technical-architecture-design.md` | 增量架构方案 |
| S3 新系统 | `docs/architecture/technical-architecture-design.md` | 完整架构方案 |
| S4 重构迁移 | `docs/architecture/migration-plan.md` + `technical-architecture-design.md` | 迁移方案 |
| S5 性能专项 | `docs/architecture/performance-optimization-plan.md` | 优化方案 |

## S1 评审版模板

```markdown
# [系统名称] 架构评审报告

## 文档信息
- 版本: v1.0
- 场景模式: S1 架构评审
- 评审日期: [日期]

## 目录
1. 评审概述与方法论
2. 系统现状概览
3. 各维度评估（业务/技术/后端/前端/数据/安全/运维）
4. 健康度评分汇总
5. 技术债务清单
6. 调整建议（按优先级排序）
7. 演进路线图建议
```

## S2 增量需求模板

```markdown
# [项目名称] 增量架构方案

## 文档信息
- 版本: v1.0
- 场景模式: S2 增量需求落地
- 基于系统: [现有系统名称/版本]
- 创建日期: [日期]

## 目录
1. 需求概述与影响面分析
2. 变更影响矩阵
3. 增量业务架构分析
4. 增量技术方案设计
5. 增量后端设计（服务/接口/数据变更）
6. 增量前端设计（页面/组件变更）
7. 增量接口契约
8. 回归风险评估与测试重点
9. 实施计划与交付
```
