# 战略分析框架（V2.0）

本文件用于将“宏观趋势判断 → 行业竞争评估 → 内部能力识别 → 战略生成与优先级排序”串联成一条可执行的分析流水线。

---

## 1. 宏观周期框架：卡尔洛塔·佩雷斯技术-经济范式

### 1.1 五次连续技术革命（用于定位历史与产业阶段）

| 技术革命 | 起始时间（常用标注） | 核心技术簇 | 基础设施 | 典型范式关键词 |
|---|---|---|---|---|
| 工业革命 | 1771 | 机械化棉纺、铁器、机械制造 | 运河、水运、公路 | 工厂化、机械化 |
| 蒸汽与铁路时代 | 1829 | 蒸汽机、铁路、煤炭、钢铁 | 全国铁路网络 | 规模扩张、市场整合 |
| 钢铁、电力与重工业时代 | 1875 | 廉价钢材、电气工程、化工 | 跨洲铁路、电网、电报 | 大企业、全球协同 |
| 石油、汽车与大规模生产时代 | 1908 | 内燃机、汽车、石化 | 高速公路、机场、电话网络 | 福特制、标准化 |
| 信息与通信技术时代 | 1971 | 微电子、软件、互联网、通信 | 数据中心、互联网、移动网络 | 网络化、柔性化、平台化 |

### 1.2 安装期 / 转折点 / 部署期判定

| 阶段 | 主导资本 | 核心特征 | 常见信号 | 战略含义 |
|---|---|---|---|---|
| 安装期 | 金融资本 | 新技术爆发、资本涌入、旧体系被冲击 | 估值狂热、资产泡沫、叙事先行 | 高探索、高不确定、快试错 |
| 转折点 | 金融与制度重构 | 泡沫出清、监管重塑、秩序重建 | 融资收缩、整合并购、政策收紧或校准 | 现金流优先、资产重估、能力重组 |
| 部署期 | 生产资本 | 技术扩散、效率提升、规模化落地 | 标准统一、应用普及、盈利改善 | 扩张应用场景、优化运营、平台协同 |

### 1.3 佩雷斯框架使用方法（分析模板）

```markdown
## 技术-经济范式定位
- 当前行业所处革命阶段：
- 当前子行业周期判断（安装期/转折点/部署期）：
- 判定证据（融资、监管、技术成熟度、盈利模式）：
- 战略启示（未来3年）：
```

## 2. 多学科洞察框架：查理·芒格思维模型

### 2.1 物理学、生物学、心理学在商业分析中的应用

| 学科镜头 | 可借用概念 | 商业问题 | 典型分析问题 |
|---|---|---|---|
| 物理学 | 临界点、杠杆、反馈回路、路径依赖 | 市场爆发点、规模效率、平台增长 | 业务增长临界规模是多少？关键杠杆在哪？ |
| 生物学 | 进化适应、生态位、共生关系 | 行业演化、生态竞争、组织韧性 | 公司在生态中的位置是否稳定？ |
| 心理学 | 激励偏差、损失厌恶、社会认同 | 用户决策、组织行为、渠道效率 | 当前激励是否导致短期主义或错误行为？ |

### 2.2 多学科交叉验证要求（最低标准）

1. 至少使用 3 个不同学科模型解释同一结论。
2. 每个结论至少有两类证据（数据证据 + 行为/案例证据）。
3. 对关键判断给出“反证路径”（若判断错误，最可能错在哪里）。

### 2.3 示例（可口可乐）

- **数学/复利**：长期品牌投入 + 分销扩张形成复利。
- **心理学/条件反射与社会认同**：品牌与愉悦场景绑定，形成心智占位。
- **生理与行为强化**：口味和消费习惯形成高复购。
- **系统杠杆**：全球灌装与渠道网络构成难复制分发系统。

## 3. 外部分析传导链：PESTLE → 五力 → SWOT(O/T)

### 3.1 传导流程（必做）

1. **PESTLE识别趋势**：列出6类因素中的关键变化。
2. **映射到五力**：判断趋势会增强/减弱哪一股力量。
3. **量化优先级**：以影响度×概率评分。
4. **归纳为O/T**：形成SWOT的机会与威胁输入。

### 3.2 传导映射模板

```markdown
## PESTLE → 五力传导表
| PESTLE趋势 | 影响五力 | 影响方向 | 影响度(1-5) | 概率(1-5) | 优先级 | 说明 |
|---|---|---|---:|---:|---:|---|
| [趋势A] | [新进入者威胁] | 增强/减弱 | 4 | 3 | 12 | [原因] |

## 五力 → SWOT(O/T)转化表
| 五力变化 | 机会O or 威胁T | O/T编号 | 证据来源 | 备注 |
|---|---|---|---|---|
| [买方议价增强] | 威胁 | T1 | [数据/访谈] | [说明] |
```

### 3.3 最佳实践

- 聚焦高优先级（建议优先处理 Top 5 因素）。
- 避免“静态五力”误区，明确趋势变化方向与时间窗口。
- O/T 必须可追溯到 PESTLE 与五力的证据链。

## 4. 内部分析双引擎：BMC + VRIO（用于SWOT的S/W）

### 4.1 商业模式画布（BMC）九模块速记

客户细分、价值主张、渠道、客户关系、收入来源、关键资源、关键活动、关键伙伴、成本结构。

### 4.2 VRIO定义与竞争含义

| 维度 | 关键问题 | 评估标准 |
|---|---|---|
| Value | 是否创造客户价值或提升效率？ | 能否抓住机会/抵御威胁 |
| Rarity | 是否稀缺？ | 竞争者是否普遍具备 |
| Inimitability | 是否难复制？ | 复制成本、路径、时间 |
| Organization | 是否被组织有效利用？ | 架构、流程、激励是否匹配 |

| V | R | I | O | 竞争含义 |
|---|---|---|---|---|
| 否 | - | - | - | 竞争劣势 |
| 是 | 否 | - | - | 竞争对等 |
| 是 | 是 | 否 | - | 暂时竞争优势 |
| 是 | 是 | 是 | 否 | 未被充分利用的优势 |
| 是 | 是 | 是 | 是 | 可持续竞争优势 |

### 4.3 BMC + VRIO联动模板

```markdown
## 内部能力盘点（来自BMC）
- 候选资源/能力1：
- 候选资源/能力2：

## VRIO评估
| 资源/能力 | V | R | I | O | 竞争含义 | SWOT归类 |
|---|---|---|---|---|---|---|
| [能力A] | 是 | 是 | 是 | 是 | 可持续优势 | S1 |
| [能力B] | 是 | 否 | - | - | 竞争对等 | W1（若缺失则劣势） |
```

## 5. 战略群组分析（竞争格局）

### 5.1 标准步骤

1. 选取2个最能区分战略差异的维度。
2. 绘制战略群组图（公司散点 + 群组圈层）。
3. 标识各群组市场规模与盈利水平。
4. 识别移动壁垒（品牌、渠道、规模、技术、监管）。
5. 提炼空白地带与跨群组威胁。

### 5.2 输出模板

```markdown
## 战略群组分析
- 维度X：
- 维度Y：
- 群组划分：
- 各群组利润特征：
- 移动壁垒：
- 机会窗口：
- 主要威胁：
```

## 6. 公司深度分析模板（目标公司/关键对手）

### 6.1 核心模块

1. **财务健康**：收入增长、毛利净利、现金流、偿债能力、营运效率。
2. **技术护城河**：专利与数据资产、研发效率、网络效应、用户转换成本。
3. **组织能力与关键人物**：管理层质量、组织执行力、人才梯队、关键人风险。

## 7. SWOT 与 TOWS 推导脚手架

### 7.1 SWOT输入原则

- **S/W** 必须来自 BMC+VRIO 证据。
- **O/T** 必须来自 PESTLE+五力+群组分析证据。
- 每项要素建议“编号 + 一句话定义 + 一条证据”。

### 7.2 TOWS矩阵脚手架（引导问题 + 填写说明）

| 组合 | 引导问题 | 填写说明 |
|---|---|---|
| SO | 如何用 S 抓住 O？ | 用“优势×机会”生成增长动作，强调速度与规模化。
| ST | 如何用 S 对冲 T？ | 用核心能力构建防御，避免被动应战。
| WO | 如何借 O 修复 W？ | 借外部窗口补短板，常见手段是合作、并购、能力建设。
| WT | 如何降低 W 与 T 的叠加损失？ | 以防守和风险控制为主，必要时收缩与剥离。

   - 负责人：
   - KPI：

## 8. 战略选项优先级矩阵与评分卡

### 8.1 评分维度建议

| 维度 | 权重建议 |
|---|---:|
| 战略重要性 | 25% |
| 资源匹配度 | 20% |
| 实施难度（易=高分） | 15% |
| 预期回报 | 20% |
| 风险暴露（低=高分） | 10% |
| 见效周期（短=高分） | 10% |

### 8.2 评分卡模板

```markdown
## 战略优先级评分卡
| 战略选项 | 重要性 | 资源匹配 | 实施难度 | 预期回报 | 风险暴露 | 见效周期 | 加权总分 | 优先级 |
|---|---:|---:|---:|---:|---:|---:|---:|---|
| 选项A | 9 | 8 | 6 | 9 | 7 | 7 | 7.95 | P0 |
| 选项B | 8 | 7 | 7 | 7 | 8 | 8 | 7.55 | P1 |
```

### 8.3 决策输出

- **Now（现在做）**：高分 + 高紧迫。
- **Next（下一步做）**：高潜力但受资源/时机限制。
- **Later/No（后做或不做）**：低回报或高风险不可控。

## 9. 战略叙事（Strategic Narrative）构建

目标：将零散分析转化为“决策者可快速理解并愿意行动”的叙事。

### 9.1 四段式结构

1. **现状（Where we are）**：行业与竞争位置。
2. **冲突（Why change now）**：关键挑战与窗口机会。
3. **抉择（What we choose）**：核心战略与取舍逻辑。
4. **承诺（How we win）**：行动路线、资源配置、里程碑。

### 9.2 叙事模板

```markdown
## 战略叙事（董事会版）

### 1) 现状
[PESTLE + 五力 + 战略群组核心结论]

### 2) 冲突
[若不行动的代价 + 当前窗口期]

### 3) 抉择
[核心战略 + 为什么是它 + 为什么不是其他选项]

### 4) 承诺
[12-24个月行动计划 + KPI + 风险对冲方案]
```
