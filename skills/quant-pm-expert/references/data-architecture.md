# 数据架构规范

## 1. 数据分层架构

```
┌─────────────────────────────────────────┐
│            应用层 (Application)          │
│  策略引擎 | 回测系统 | 风控系统 | 报表    │
├─────────────────────────────────────────┤
│            服务层 (Service)              │
│  数据API | 因子服务 | 信号服务 | 订阅    │
├─────────────────────────────────────────┤
│            计算层 (Compute)              │
│  因子计算 | 指标计算 | 特征工程 | 聚合    │
├─────────────────────────────────────────┤
│            存储层 (Storage)              │
│  时序库 | 关系库 | 缓存 | 对象存储       │
├─────────────────────────────────────────┤
│            采集层 (Collection)           │
│  行情接口 | 财务接口 | 另类数据 | 爬虫    │
└─────────────────────────────────────────┘
```

## 2. 数据源接入

### 2.1 行情数据源

| 数据源 | 数据类型 | 延迟 | 成本 |
|--------|----------|------|------|
| 交易所直连 | Tick | <1ms | 高 |
| 券商API | 分钟线 | <1s | 中 |
| 第三方数据商 | 日线 | 分钟级 | 低 |

### 2.2 基本面数据源

| 数据源 | 数据类型 | 更新频率 |
|--------|----------|----------|
| Wind | 财务/估值 | 日度 |
| 同花顺iFind | 财务/行业 | 日度 |
| 东方财富Choice | 财务/资讯 | 日度 |

## 3. 数据存储方案

### 3.1 存储选型

| 数据类型 | 推荐存储 | 原因 |
|----------|----------|------|
| Tick数据 | InfluxDB/QuestDB | 高写入性能 |
| 分钟/日线 | ClickHouse | 高压缩比 |
| 财务数据 | PostgreSQL | 关系查询 |
| 因子数据 | Redis + Parquet | 快速读取 |

### 3.2 数据压缩

| 压缩方式 | 压缩比 | 适用场景 |
|----------|--------|----------|
| LZ4 | 2-3x | 实时数据 |
| ZSTD | 4-6x | 历史数据 |
| Parquet | 5-10x | 分析数据 |

## 4. 数据质量管理

### 4.1 质量检查项

| 检查项 | 方法 | 处理方式 |
|--------|------|----------|
| 缺失值 | 空值检测 | 插值/剔除 |
| 异常值 | 3σ原则 | 修正/标记 |
| 重复值 | 去重检测 | 保留最新 |
| 一致性 | 跨源比对 | 以权威源为准 |
