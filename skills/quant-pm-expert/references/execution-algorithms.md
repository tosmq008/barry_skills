# 执行算法规范

## 1. 算法交易概述

### 1.1 执行目标

| 目标 | 描述 | 度量指标 |
|------|------|----------|
| 最小化冲击 | 减少市场影响 | 实现差额 |
| 最优执行 | 获得最佳价格 | 滑点 |
| 隐蔽执行 | 隐藏交易意图 | 信息泄露 |

## 2. 常用执行算法

### 2.1 TWAP (时间加权)

| 参数 | 说明 | 建议值 |
|------|------|--------|
| 执行时长 | 总执行时间 | 30-120分钟 |
| 切片间隔 | 每次下单间隔 | 1-5分钟 |
| 切片大小 | 每次下单量 | 总量/切片数 |

**适用场景：** 流动性一般、无紧迫性

### 2.2 VWAP (成交量加权)

| 参数 | 说明 | 建议值 |
|------|------|--------|
| 历史窗口 | 参考成交量分布 | 20日 |
| 参与率 | 占市场成交比例 | 5-20% |
| 偏离容忍 | 允许偏离度 | ±10% |

**适用场景：** 跟随市场节奏、大单执行

### 2.3 POV (参与率)

| 参数 | 说明 | 建议值 |
|------|------|--------|
| 目标参与率 | 占市场比例 | 10-30% |
| 最小切片 | 最小下单量 | 1000股 |
| 价格限制 | 价格偏离限制 | ±0.5% |

**适用场景：** 控制市场冲击

## 3. 滑点控制

### 3.1 滑点来源

| 来源 | 原因 | 控制方法 |
|------|------|----------|
| 买卖价差 | 流动性成本 | 选择流动性好的标的 |
| 市场冲击 | 订单影响价格 | 拆单执行 |
| 时间延迟 | 系统延迟 | 优化系统架构 |
| 信息泄露 | 意图暴露 | 隐蔽执行 |

### 3.2 滑点预估

| 订单规模 | 预估滑点 |
|----------|----------|
| <日成交1% | 0.1-0.2% |
| 1-5% | 0.2-0.5% |
| 5-10% | 0.5-1.0% |
| >10% | >1.0% |

## 4. 订单类型选择

| 订单类型 | 优点 | 缺点 | 适用场景 |
|----------|------|------|----------|
| 市价单 | 确保成交 | 价格不确定 | 紧急交易 |
| 限价单 | 价格可控 | 可能不成交 | 常规交易 |
| 冰山单 | 隐藏规模 | 执行较慢 | 大单 |
| 条件单 | 自动触发 | 需要监控 | 策略执行 |

## 5. 执行质量评估

| 指标 | 计算方法 | 优秀标准 |
|------|----------|----------|
| 实现差额 | 实际价-决策价 | <0.2% |
| VWAP偏离 | 实际价-VWAP | <0.1% |
| 完成率 | 成交量/目标量 | >95% |
| 执行时间 | 实际/计划时间 | ±10% |
