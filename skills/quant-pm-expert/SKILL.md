# 量化产品经理专家 Skill

作为量化交易领域的资深产品经理，运用专业的量化分析框架和产品设计方法论，对量化交易系统进行系统性的功能设计、数据架构规划和策略产品化设计。

## When to Use

**适用场景：**
- 设计量化交易系统的产品功能
- 规划量化策略的产品化路径
- 设计数据分析和回测系统
- 制定风险管理产品方案
- 评估量化系统的性能指标
- 设计交易信号和执行系统
- 规划量化平台的用户体验

**不适用：**
- 纯技术实现（代码开发）
- 具体策略的数学推导
- 实时交易执行操作

## Phase 2: 策略产品化 (Strategy Productization)

> ⚠️ **执行前必须读取 `references/strategy-design.md` 获取策略设计规范**

### 2.1 策略类型体系

| 策略类型 | 描述 | 适用市场 | 复杂度 |
|----------|------|----------|--------|
| 趋势跟踪 | 追随价格趋势 | 趋势明显市场 | 低 |
| 均值回归 | 价格偏离后回归 | 震荡市场 | 中 |
| 动量策略 | 强者恒强 | 牛市 | 中 |
| 多因子模型 | 多维度选股 | 全市场 | 高 |
| 统计套利 | 价差收敛 | 配对交易 | 高 |
| 机器学习 | 数据驱动预测 | 全市场 | 极高 |

### 2.2 因子库设计

**因子分类体系：**

| 因子大类 | 子类 | 示例因子 |
|----------|------|----------|
| 估值因子 | 价值型 | PE、PB、PS、PCF |
| 成长因子 | 成长型 | 营收增长率、净利润增长率 |
| 质量因子 | 盈利能力 | ROE、ROA、毛利率 |
| 动量因子 | 价格动量 | 20日涨幅、相对强弱 |
| 波动因子 | 风险度量 | 波动率、Beta |
| 流动性因子 | 交易活跃度 | 换手率、成交额 |
| 技术因子 | 技术分析 | MACD、RSI、布林带 |
| 情绪因子 | 市场情绪 | 融资余额、北向资金 |

### 2.3 策略评估标准

| 评估维度 | 优秀 | 合格 | 不合格 |
|----------|------|------|--------|
| 年化收益 | >20% | 10-20% | <10% |
| 夏普比率 | >2.0 | 1.0-2.0 | <1.0 |
| 最大回撤 | <10% | 10-20% | >20% |
| 胜率 | >55% | 45-55% | <45% |
| 盈亏比 | >2.0 | 1.5-2.0 | <1.5 |

### 2.4 机器学习策略设计

| 模型类型 | 应用场景 | 特征工程 | 注意事项 |
|----------|----------|----------|----------|
| 线性回归 | 因子收益预测 | 因子标准化 | 防止过拟合 |
| 随机森林 | 涨跌分类 | 特征选择 | 样本平衡 |
| LSTM | 时序预测 | 序列构造 | 数据量要求高 |
| XGBoost | 多因子选股 | 特征重要性 | 参数调优 |
| 强化学习 | 动态仓位 | 状态定义 | 训练不稳定 |

### 2.5 Alpha因子研究流程 (业界标准)

> 参考BlackRock、Goldman Sachs等顶级机构的因子研究方法论

```
┌─────────────────────────────────────────────────────────────────┐
│                    Alpha因子研究流程                              │
├─────────────────────────────────────────────────────────────────┤
│  假设提出 ──▶ 因子构建 ──▶ 单因子检验 ──▶ 因子合成 ──▶ 组合构建 │
│     │           │            │            │            │        │
│  经济逻辑    数学表达     IC/IR分析    权重优化     风险控制     │
│  文献支撑    标准化处理   分组回测     正交化       再平衡       │
│  数据验证    去极值       衰减分析     多空组合     交易成本     │
└─────────────────────────────────────────────────────────────────┘
```

**因子检验指标：**

| 指标 | 计算方法 | 优秀标准 | 说明 |
|------|----------|----------|------|
| IC均值 | corr(因子, 下期收益) | > 0.03 | 预测能力 |
| IC_IR | IC均值/IC标准差 | > 0.5 | 稳定性 |
| 多空收益 | Top组-Bottom组 | > 10%年化 | 区分度 |
| 换手率 | 月度调仓比例 | < 30% | 交易成本 |
| 因子衰减 | IC随滞后期变化 | 慢衰减 | 持续性 |

### 2.6 过拟合检测与防范

| 检测方法 | 描述 | 判断标准 |
|----------|------|----------|
| 样本外测试 | 留出30%数据验证 | 样本外表现≥80%样本内 |
| 滚动回测 | 滚动窗口前推 | 各窗口表现稳定 |
| 参数敏感性 | 微调参数观察变化 | 收益变化<20% |
| 多市场验证 | 跨市场/跨时段测试 | 多数市场有效 |
| 蒙特卡洛 | 随机打乱数据 | 真实>95%随机 |
| 白噪声测试 | 用随机信号回测 | 真实显著优于随机 |

### 2.7 策略容量评估

| 容量因素 | 影响 | 评估方法 |
|----------|------|----------|
| 市场流动性 | 大资金难以进出 | 日均成交额分析 |
| 冲击成本 | 大单影响价格 | 冲击成本模型 |
| 策略拥挤度 | 同质策略竞争 | 因子拥挤度指标 |
| 换手率 | 高换手消耗容量 | 换手率×冲击成本 |

**容量估算公式：**
```
策略容量 ≈ 日均成交额 × 参与率上限 × 持仓周期 / 换手率
```

## Phase 4: 回测系统设计 (Backtesting System)

> ⚠️ **执行前必须读取 `references/backtest-framework.md` 获取回测框架规范**

### 4.1 回测引擎架构

**核心组件：**

| 组件 | 功能 | 关键指标 |
|------|------|----------|
| 事件引擎 | 驱动回测流程 | 事件处理延迟 < 1ms |
| 数据馈送 | 提供历史数据 | 支持Tick/分钟/日线 |
| 策略容器 | 运行策略逻辑 | 支持多策略并行 |
| 撮合引擎 | 模拟订单成交 | 支持多种撮合模式 |
| 绩效分析 | 计算回测指标 | 50+绩效指标 |

### 4.2 绩效指标体系

**收益指标：**
| 指标 | 公式/说明 | 健康值 |
|------|-----------|--------|
| 年化收益率 | (1+总收益)^(252/交易日)-1 | > 15% |
| 累计收益率 | (期末净值-1)*100% | 正值 |
| 超额收益 | 策略收益 - 基准收益 | > 0 |

**风险指标：**
| 指标 | 公式/说明 | 健康值 |
|------|-----------|--------|
| 最大回撤 | 峰值到谷值的最大跌幅 | < 20% |
| 波动率 | 收益率标准差年化 | < 25% |
| 下行波动率 | 负收益的标准差 | < 15% |

**风险调整收益：**
| 指标 | 公式/说明 | 健康值 |
|------|-----------|--------|
| 夏普比率 | (收益-无风险)/波动率 | > 1.5 |
| 索提诺比率 | (收益-无风险)/下行波动 | > 2.0 |
| 卡玛比率 | 年化收益/最大回撤 | > 1.0 |

### 4.3 回测陷阱与实盘差异

> 业界经验：回测盈利的策略80%在实盘中失效

**常见回测陷阱：**

| 陷阱类型 | 描述 | 防范措施 |
|----------|------|----------|
| 未来函数 | 使用未来数据 | 严格时间戳检查 |
| 幸存者偏差 | 只用存活股票 | 使用全历史数据 |
| 前视偏差 | 用未来信息选股 | 延迟数据发布 |
| 过度拟合 | 参数过度优化 | 样本外验证 |
| 忽略成本 | 不计交易成本 | 保守估计成本 |
| 流动性假设 | 假设无限流动性 | 成交量限制 |

**回测与实盘差异来源：**

| 差异来源 | 回测假设 | 实盘现实 | 影响程度 |
|----------|----------|----------|----------|
| 成交价格 | 收盘价成交 | 滑点存在 | 高 |
| 成交量 | 无限流动性 | 有限流动性 | 高 |
| 执行延迟 | 即时执行 | 网络延迟 | 中 |
| 市场冲击 | 无冲击 | 大单冲击 | 高 |
| 数据质量 | 完美数据 | 数据错误 | 中 |

## Phase 5: 风险管理设计 (Risk Management)

> ⚠️ **执行前必须读取 `references/risk-management.md` 获取风险管理规范**

### 5.1 风控指标体系

| 风控维度 | 指标 | 阈值设置 | 触发动作 |
|----------|------|----------|----------|
| 单笔风险 | 单笔亏损比例 | < 2% | 强制止损 |
| 日度风险 | 日亏损上限 | < 5% | 暂停交易 |
| 持仓风险 | 单股持仓上限 | < 20% | 禁止加仓 |
| 行业风险 | 行业集中度 | < 30% | 分散配置 |
| 系统风险 | Beta暴露 | < 1.2 | 对冲调整 |

### 5.2 仓位管理模型

**常用仓位管理方法：**

| 方法 | 描述 | 适用场景 |
|------|------|----------|
| 固定比例法 | 每笔固定资金比例 | 初学者 |
| 凯利公式 | 基于胜率和赔率 | 高频交易 |
| 波动率调整 | 根据波动率动态调整 | 趋势策略 |
| 风险平价 | 各资产风险贡献相等 | 资产配置 |
| ATR仓位 | 基于ATR计算仓位 | 趋势跟踪 |

### 5.3 止损止盈机制

| 类型 | 方法 | 参数设置 |
|------|------|----------|
| 固定止损 | 固定百分比 | -5% ~ -8% |
| 移动止损 | 跟踪最高点 | 回撤3% ~ 5% |
| 时间止损 | 持仓时间限制 | 5 ~ 20个交易日 |
| 波动止损 | 基于ATR | 2 ~ 3倍ATR |
| 技术止损 | 突破关键位 | 支撑/阻力位 |

### 5.4 行为金融与情绪管理

> 量化的核心价值：用规则替代情绪决策

**常见认知偏差及应对：**

| 认知偏差 | 表现 | 量化解决方案 |
|----------|------|--------------|
| 损失厌恶 | 不愿止损 | 强制止损规则 |
| 过度自信 | 重仓押注 | 仓位上限控制 |
| 锚定效应 | 执着成本价 | 基于信号决策 |
| 羊群效应 | 追涨杀跌 | 逆向/动量规则 |
| 近因偏差 | 过度反应近期 | 长周期回测验证 |
| 确认偏差 | 只看支持信息 | 多维度因子验证 |

**情绪隔离设计原则：**
1. **规则先行** - 所有决策必须有预设规则
2. **自动执行** - 减少人工干预环节
3. **延迟确认** - 重大决策设置冷静期
4. **日志审计** - 记录所有手动干预并复盘

### 6.3 市场微观结构考量

> 高频/算法交易必须理解的市场微观结构

## Best Practices 最佳实践

### 策略开发最佳实践

1. **先验证假设再开发** - 用统计方法验证策略逻辑
2. **样本外测试必不可少** - 防止过拟合
3. **考虑交易成本** - 滑点、手续费、冲击成本
4. **简单策略优先** - 复杂不等于有效

### 风险管理最佳实践

1. **永远设置止损** - 没有止损的策略不是好策略
2. **分散投资** - 不要把鸡蛋放在一个篮子里
3. **仓位控制** - 单笔风险不超过总资金2%
4. **定期复盘** - 分析亏损原因，持续改进

### 系统设计最佳实践

1. **数据质量第一** - 垃圾进垃圾出
2. **模块化设计** - 策略、数据、执行分离
3. **充分测试** - 回测、模拟盘、小资金实盘
4. **监控告警** - 7x24小时监控系统状态
