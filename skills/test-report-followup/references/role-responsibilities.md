# 角色职责说明

## 角色概览

| 角色 | 主要职责 | 处理问题类型 |
|------|----------|--------------|
| 产品经理 | 需求澄清、功能评估 | PRD问题、缺失功能、体验问题 |
| UI设计师 | 设计修复、体验优化 | UI Bug、设计问题、体验问题 |
| 前端工程师 | 前端代码修复 | 功能Bug、UI实现、交互问题 |
| 后端工程师 | 后端代码修复 | API Bug、数据问题、性能问题 |
| 测试专家 | 回归验证 | 所有问题的验证 |

---

## 产品经理 (Product Manager)

### 职责范围

```
┌─────────────────────────────────────────────────────────────────┐
│                      产品经理职责                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. PRD 问题处理                                                │
│     ├── 澄清需求描述不清的问题                                  │
│     ├── 解决需求逻辑矛盾                                        │
│     ├── 补充缺失的需求说明                                      │
│     └── 更新 PRD 文档                                           │
│                                                                 │
│  2. 缺失功能评估                                                │
│     ├── 评估功能重要性                                          │
│     ├── 确定功能优先级                                          │
│     ├── 决定是否纳入当前版本                                    │
│     └── 提供简要需求说明                                        │
│                                                                 │
│  3. 体验问题决策                                                │
│     ├── 确认体验问题的改进方向                                  │
│     ├── 与设计师沟通改进方案                                    │
│     └── 确定改进优先级                                          │
│                                                                 │
│  4. 跨角色协调                                                  │
│     ├── 协调设计与开发的沟通                                    │
│     └── 解决需求理解分歧                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 输出物

| 输出物 | 说明 | 格式 |
|--------|------|------|
| 需求澄清文档 | 对不清晰需求的详细说明 | Markdown |
| PRD 更新 | 更新后的 PRD 文档 | Markdown |
| 功能评估报告 | 缺失功能的优先级评估 | Markdown |
| 决策记录 | 重要决策的记录 | Markdown |

### 响应时限

| 优先级 | 响应时限 | 说明 |
|--------|----------|------|
| P0 | 2小时内 | 阻塞性问题 |
| P1 | 4小时内 | 影响主流程 |
| P2 | 1天内 | 一般问题 |
| P3 | 3天内 | 低优先级 |

## UI设计师 (UI Designer)

### 职责范围

```
┌─────────────────────────────────────────────────────────────────┐
│                      UI设计师职责                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. UI Bug 修复                                                 │
│     ├── 分析 UI 问题原因                                        │
│     ├── 提供修复后的设计稿                                      │
│     ├── 提供设计标注                                            │
│     └── 确认前端实现效果                                        │
│                                                                 │
│  2. 设计问题处理                                                │
│     ├── 补充缺失的设计稿                                        │
│     ├── 修正设计不合理之处                                      │
│     └── 提供设计规范说明                                        │
│                                                                 │
│  3. 体验优化                                                    │
│     ├── 根据产品方向设计优化方案                                │
│     ├── 提供交互优化建议                                        │
│     └── 输出优化后的设计稿                                      │
│                                                                 │
│  4. 设计验收                                                    │
│     ├── 验收前端 UI 实现                                        │
│     └── 确认与设计稿一致                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 输出物

| 输出物 | 说明 | 格式 |
|--------|------|------|
| 修复设计稿 | 修复后的 UI 设计 | Figma/Sketch |
| 设计标注 | 详细的设计标注 | 标注工具 |
| 设计规范 | 设计规范说明 | Markdown |
| 交互说明 | 交互逻辑说明 | Markdown |

### 响应时限

| 优先级 | 响应时限 | 说明 |
|--------|----------|------|
| P0 | 4小时内 | 严重 UI 问题 |
| P1 | 8小时内 | 影响体验 |
| P2 | 2天内 | 一般问题 |
| P3 | 1周内 | 优化类 |

## 沟通规范

### 任务分派沟通

```markdown
## 任务分派通知

@[角色] 您好，

根据测试报告 test-report-v[x.x.x].md，有以下任务需要您处理：

### 任务清单
| 问题ID | 标题 | 优先级 | 截止日期 |
|--------|------|--------|----------|
| [ID] | [标题] | [优先级] | [日期] |

### 详细任务单
请查看：docs/test/followup/task-[角色].md

### 注意事项
- P0/P1 任务请优先处理
- 完成后请更新任务状态
- 如有问题请及时沟通

谢谢！
```
