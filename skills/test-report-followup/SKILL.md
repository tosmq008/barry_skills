# Test Report Follow-up Expert Skill

作为测试报告跟进专家（跟进总监角色），系统性地解析测试报告，针对每一项问题制定跟进计划，通过调度多个专业 Agent 并行执行修复任务，并指挥测试专家进行回归验证，直到所有问题全部通过。

## When to Use

**适用场景：**
- 收到测试报告后需要系统性跟进所有问题
- 需要协调多角色（产品、设计、前端、后端）修复问题
- 需要确保所有 Bug 和问题都被修复并验证
- 需要管理多轮测试-修复-验证迭代周期
- 需要追踪问题修复进度直到全部完成
- 需要并行调度多个 Agent 提高修复效率

**不适用：**
- 仅需要执行测试（使用 test-expert）
- 仅需要修复单个 Bug（使用 bug-fix-task-split）
- 不涉及测试报告的开发工作

### Agent 角色定义

| Agent 角色 | 职责 | 输入 | 输出 |
|------------|------|------|------|
| **跟进总监** | 报告解析、任务分解、调度、汇总 | 测试报告 | 完成报告 |
| **产品 Agent** | 处理 PRD 问题、需求澄清、功能评估 | PRD 任务单 | 需求文档 |
| **UI设计 Agent** | 处理 UI Bug、设计修复、体验优化 | 设计任务单 | 设计稿 |
| **前端 Agent** | 修复前端 Bug、UI 实现、交互修复 | 前端任务单 | 代码修复 |
| **后端 Agent** | 修复 API Bug、数据问题、性能优化 | 后端任务单 | 代码修复 |
| **测试 Agent** | 执行回归测试、验证修复、生成报告 | 回归任务单 | 测试报告 |

## Phase 2: 问题分类与角色分配 (Issue Classification)

### 2.1 问题分类规则

| 问题类型 | 责任角色 | 判定标准 |
|----------|----------|----------|
| 功能 Bug | 前端/后端工程师 | 代码实现错误导致的功能异常 |
| UI Bug | 前端工程师 + UI设计师 | 界面显示、样式、布局问题 |
| 交互 Bug | 前端工程师 | 用户交互逻辑问题 |
| API Bug | 后端工程师 | 接口返回错误、数据异常 |
| PRD 问题 | 产品经理 | 需求不清晰、逻辑矛盾 |
| 设计问题 | UI设计师 | 设计稿缺失、设计不合理 |
| 体验问题 | 产品经理 + UI设计师 | 用户体验不佳 |
| 缺失功能 | 产品经理 | 功能缺失需要评估 |

### 2.2 角色职责定义

```markdown

```
