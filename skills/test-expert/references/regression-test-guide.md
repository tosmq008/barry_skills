# 回归测试指南

## 回归测试概述

回归测试是在软件修改后，验证原有功能是否仍然正常工作的测试活动。

## 回归测试触发条件

| 触发条件 | 回归范围 | 版本号变更 |
|----------|----------|------------|
| Bug 修复 | 相关 Bug + 影响范围用例 | 修订号 +1 (v1.0.x) |
| 功能迭代 | 全量回归 + 新功能测试 | 次版本 +1 (v1.x.0) |
| 重大重构 | 全量回归测试 | 主版本 +1 (vx.0.0) |

---

## 回归测试执行流程

### Step 1: 检查历史报告

```bash
# 检查测试报告目录
ls -la docs/test/test-report-*.md

# 找到最新版本
ls -t docs/test/test-report-*.md | head -1
```

### Step 2: 解析历史报告

**必须提取的信息：**

1. **Bug 列表**
   - 所有 Open 状态的 Bug
   - 所有 Fixed 待验证的 Bug
   - 所有遗留问题

2. **失败用例**
   - 所有 Fail 状态的用例
   - 所有 Block 状态的用例

3. **通过用例**
   - P0 级别用例（必须回归）
   - P1 级别用例（建议回归）

### Step 3: 制定回归计划

```markdown
# 回归测试计划

## 回归范围
- 上版本: v[x.x.x]
- 本版本: v[x.x.x]
- 回归类型: Bug修复回归 / 功能迭代回归 / 全量回归

## 回归项清单

### 必须回归 (Must)
| 类型 | ID | 描述 | 优先级 |
|------|-----|------|--------|
| Bug | BUG_001 | [描述] | P0 |
| 用例 | TC_001 | [描述] | P0 |

### 建议回归 (Should)
| 类型 | ID | 描述 | 优先级 |
|------|-----|------|--------|
| 用例 | TC_010 | [描述] | P1 |

### 可选回归 (Could)
| 类型 | ID | 描述 | 优先级 |
|------|-----|------|--------|
| 用例 | TC_020 | [描述] | P2 |

## 预计工时
- 必须回归: [n] 小时
- 建议回归: [n] 小时
- 总计: [n] 小时
```

### Step 4: 执行回归测试

**执行顺序：**
1. Bug 回归验证（按优先级）
2. 失败用例回归
3. P0 用例回归
4. P1 用例抽样回归

### Step 5: 生成回归报告

## Bug 回归验证规范

### 验证流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      Bug 回归验证流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 读取原始 Bug 信息                                           │
│     ├── Bug ID、标题、严重程度                                  │
│     ├── 复现步骤                                                │
│     ├── 复现环境                                                │
│     └── 测试数据                                                │
│                                                                 │
│  2. 准备验证环境                                                │
│     ├── 确认版本号                                              │
│     ├── 准备测试数据                                            │
│     └── 配置测试环境                                            │
│                                                                 │
│  3. 按原步骤复现                                                │
│     ├── 严格按照原复现步骤执行                                  │
│     ├── 记录每一步的实际结果                                    │
│     └── 截图保存证据                                            │
│                                                                 │
│  4. 判定验证结果                                                │
│     ├── ✅ 已修复: Bug 不再复现                                 │
│     ├── ❌ 未修复: Bug 仍然存在                                 │
│     ├── ⚠️ 部分修复: 问题部分解决                              │
│     └── 🔄 引入新问题: 修复导致新 Bug                          │
│                                                                 │
│  5. 记录验证结果                                                │
│     └── 填写回归验证记录                                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 验证结果判定标准

| 结果 | 判定标准 | 后续动作 |
|------|----------|----------|
| ✅ 已修复 | 按原步骤无法复现问题 | 关闭 Bug，标记为 Verified |
| ❌ 未修复 | 按原步骤仍可复现问题 | 重新打开 Bug，通知开发 |
| ⚠️ 部分修复 | 主要问题解决，但有残留 | 更新 Bug 描述，降低优先级 |
| 🔄 引入新问题 | 修复导致新的问题 | 创建新 Bug，关联原 Bug |

## 用例回归验证规范

### 回归策略

| 用例优先级 | 回归策略 | 说明 |
|------------|----------|------|
| P0 | 100% 回归 | 核心功能必须全部验证 |
| P1 | 80% 回归 | 重要功能大部分验证 |
| P2 | 30% 回归 | 一般功能抽样验证 |
| P3 | 按需回归 | 边缘功能按需验证 |

### 回归执行记录

```markdown
## 用例回归执行记录

### 执行信息
- 执行日期: [日期]
- 执行版本: v[x.x.x]
- 执行人: 测试专家

### 执行结果
| 用例ID | 用例名称 | 优先级 | 上次结果 | 本次结果 | 备注 |
|--------|----------|--------|----------|----------|------|
| TC_001 | [名称] | P0 | Fail | Pass | 关联 BUG_001 |
| TC_002 | [名称] | P0 | Pass | Pass | 回归通过 |
| TC_003 | [名称] | P1 | Fail | Fail | 问题未修复 |

### 统计
- 总执行: [n]
- 通过: [n]
- 失败: [n]
- 通过率: [%]
```

## 版本号管理规范

### 版本号格式

```
v[主版本].[次版本].[修订号]

示例:
v1.0.0  - 首次全量测试
v1.0.1  - 第一次 Bug 修复回归
v1.0.2  - 第二次 Bug 修复回归
v1.1.0  - 功能迭代测试
v2.0.0  - 重大版本测试
```

### 版本号递增规则

| 变更类型 | 版本号变更 | 示例 |
|----------|------------|------|
| Bug 修复 | 修订号 +1 | v1.0.0 → v1.0.1 |
| 小功能迭代 | 次版本 +1，修订号归零 | v1.0.3 → v1.1.0 |
| 大功能迭代 | 次版本 +1，修订号归零 | v1.2.1 → v1.3.0 |
| 重大重构 | 主版本 +1，其他归零 | v1.3.2 → v2.0.0 |

### 报告文件命名

```
docs/test/
├── test-report-v1.0.0.md      # 首次全量测试
├── test-report-v1.0.1.md      # Bug 修复回归
├── test-report-v1.0.2.md      # 二次回归
├── test-report-v1.1.0.md      # 功能迭代
├── regression-report-v1.0.1.md # 回归专项报告
└── test-report-latest.md      # 最新报告（副本）
```

## 回归测试检查清单

### 执行前检查
- [ ] 确认存在历史测试报告
- [ ] 确认当前版本号
- [ ] 提取所有待回归项
- [ ] 制定回归计划

### 执行中检查
- [ ] 按优先级顺序执行
- [ ] 每项都有执行记录
- [ ] 截图保存证据
- [ ] 及时记录新发现问题

### 执行后检查
- [ ] 所有回归项都已验证
- [ ] 回归报告版本号正确
- [ ] 历史版本索引已更新
- [ ] 最新报告副本已更新
