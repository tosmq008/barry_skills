# PRD Review Checklist

## 1. 完整性 (Mandatory)
- [ ] PRD文档是否有明确的背景、目标与用户场景、操作交互、功能逻辑以及业务指标
- [ ] PRD文档是否考虑到了所有用户角色和操作场景
- [ ] PRD/Prototype是否明确标注了交互流程和状态变化

## 2. 适用范围 (Mandatory)
- [ ] PRD文档中是否明确了适用的国家以及平台

## 3. 素材完整性
- [ ] PRD中涉及的素材或者附件是否完整
- [ ] PRD涉及到的消息链路是否完整（sms、push、email等）
- [ ] PRD涉及到的交互、展示、提示文案、多语言文本是否完整
- [ ] PRD涉及到的交互字段是否有对应的数据字典
- [ ] BI统计的业务变化数据是否明确

## 4. 功能复用 (Mandatory)
- [ ] 该方案是否只适用于单个国家
- [ ] 明确理由为什么其他国家不适用
- [ ] 是否分析了国家现状的逻辑
- [ ] 所有产品险种是否支持/是否定制化/明确理由
- [ ] 所有终端是否支持/是否定制化/明确理由

## 5. 流程和规则 (Mandatory)
- [ ] PRD方案业务的准入条件
- [ ] 业务链路数据来源是否明确
- [ ] 正向操作链路是否完整
- [ ] 权限描述是否完整
- [ ] 操作异常链路是否完整
- [ ] 异常引导是否清晰并闭环
- [ ] 逆向链路是否完整
- [ ] 历史版本兼容性考虑
- [ ] 边界条件和异常情况处理
- [ ] 风控规则
- [ ] 需执行的测试种类

## 6. 数据 (Mandatory)
- [ ] 所有字段是否都明确数据来源和处理逻辑
- [ ] 金额展示是否有明确计算公式

## 7. 性能指标与业务数据 (Mandatory)
- [ ] 业务吞吐量是否明确评估
- [ ] SLA要求描述
- [ ] 3年内业务规模评估
- [ ] 数据保留时效

## 8. 数据埋点 (Mandatory)
- [ ] 是否有需要后端数据埋点

## 9. 发布描述 (Mandatory)
- [ ] 发布上线计划链路是否完整
- [ ] 开关要求是否明确
- [ ] 预发布环境验证和回滚方案
- [ ] 发布先后顺序

## 10. 技术预研 (Mandatory)
- [ ] 数据是否可获得
- [ ] 系统框架是否支持
- [ ] 第三方依赖是否可行
