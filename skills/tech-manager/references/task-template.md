# 任务单模板

## 概述

本文档提供技术经理调度各专家 Agent 时使用的任务单模板。所有任务单均基于 product-expert 输出的结构化 PRD（L1/L2/L3 YAML）生成。

**PRD 来源文件：**
- `L1-feature-architecture.yaml` — 功能架构（endpoints、优先级、依赖）
- `L2-use-case-flows.yaml` — 用例流（main_flow、data_changes、business_rules）
- `L3-user-stories.yaml` — User Story + AC（Given-When-Then）

## 5. 多端联调任务单模板

> 适用于多端项目的接口联调，覆盖客户端、Web前端、后端之间的数据流验证

```markdown
# 多端联调任务单

## 基本信息
- **任务ID:** TASK_INT_[xxx]
- **来源PRD:** docs/prd/L2-use-case-flows.yaml
- **联调环境:** [环境地址]
- **涉及端:** [Client(原生)/Client(Web)/Admin/Backend]

## 关联PRD
- **L1 功能:** [F-001, F-002]
- **L2 用例:** [UC-001, UC-002]

## 联调范围

### 接口列表（各端与后端的对接）
| 序号 | 接口路径 | 客户端页面 | Web前端页面 | Admin页面 | 后端服务 |
|------|----------|-----------|-------------|-----------|----------|
| 1 | /api/v1/xxx | [页面] | [页面] | [页面] | [服务] |

### 数据流
| 序号 | 数据流向 | 涉及端 | 说明 |
|------|----------|--------|------|
| 1 | Client → Backend | 客户端/后端 | [说明] |
| 2 | Admin → Backend | Web前端/后端 | [说明] |
| 3 | Backend → Client | 后端/客户端 | [说明] |

## 各端联调检查

### 客户端 ↔ 后端（如适用）
- [ ] 接口连通性（网络/协议/端口）
- [ ] 请求参数格式正确（JSON序列化）
- [ ] 响应数据解析正确
- [ ] 认证Token传递正确
- [ ] 错误响应处理正确
- [ ] 分页/筛选参数正确

### Web前端 ↔ 后端（如适用）
- [ ] 接口连通性
- [ ] CORS 跨域配置正确
- [ ] 请求参数正确
- [ ] 响应数据正确
- [ ] 错误处理正确
- [ ] 分页/筛选正确

### 客户端 ↔ Web前端（混合方案，如适用）
- [ ] JSBridge 双向通信正常
- [ ] 原生页面与H5页面互跳正常
- [ ] 登录态共享正确
- [ ] WebView 性能达标

## 多端一致性检查（如适用）
- [ ] 同一接口在各端返回数据一致
- [ ] 核心功能各端行为一致
- [ ] 错误提示各端一致
- [ ] 状态同步各端正确

## 功能完整性检查（对照L3 AC）
| Story ID | AC-ID | 功能点 | 客户端 | Web前端 | Admin | 后端 | 联调状态 |
|----------|-------|--------|--------|---------|-------|------|----------|
| US-001 | AC-001 | [功能] | [✓/✗/-] | [✓/✗/-] | [✓/✗/-] | [✓/✗] | [✓/✗] |

## 输出要求
- 联调检查报告（各端分别记录）
- 问题列表（标注责任端和责任Agent）
- 多端一致性报告（如适用）
- 联调通过确认
```
