# 前后端多端联调检查清单

## 概述

联调不是“页面能点通”就结束，而是要确认 **架构契约、接口契约、数据流、认证、错误处理、多端一致性** 都已闭环。

在进入本清单前，必须先确认 `tech-manager` 已通过架构关卡。

## 0. 联调前前置检查

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 架构主文档已确认 | 当前实现基于稳定架构方案推进 | [ ] |
| API 契约已确认 | `docs/architecture/api-contract.md` 为唯一可信接口来源 | [ ] |
| 数据字典已确认 | `docs/architecture/data-dictionary.md` 已覆盖当前需求 | [ ] |
| 任务拆解已确认 | `docs/dev/task-breakdown.md` 已明确依赖和范围 | [ ] |
| 需要的环境已准备 | 前端、客户端、后端、测试环境可用 | [ ] |

如果以上任一项未满足，不进入联调。

## 7. 联调结论

联调报告建议至少包含以下结论：

- 通过范围
- 未通过项
- 阻塞问题
- 是否属于实现问题或架构问题
- 是否可进入测试验收

如果联调问题本质上属于架构缺口，返回 `tech-manager` 的 **Phase 0 / 架构调整清单**，而不是继续硬修实现。
