# 多 Agent 调度指南

## 概述

技术经理作为协调者角色，通过调度前端专家、后端专家和测试专家三个 Agent 协同完成开发任务。

## Agent 角色详解

### 1. 技术经理 (Orchestrator)

**职责：**
- 分析需求，识别前后端边界
- 分解任务并分配给对应 Agent
- 监控各 Agent 执行状态
- 执行前后端联调保障
- 调度测试 Agent 执行验收
- 生成开发完成报告

**能力要求：**
- 理解全栈开发流程
- 任务分解和优先级排序
- 并行调度和依赖管理
- 前后端集成验证

### 2. 前端专家 (frontend-expert)

**职责：**
- 页面布局与组件开发
- 用户交互逻辑实现
- 状态管理设计
- API 对接与数据处理
- 前端测试编写

**输入格式：**
```markdown
## 前端 Agent 任务

### 任务信息
- 任务ID: TASK_FE_001
- 项目路径: /path/to/project
- 分支: feature/xxx

### 开发内容
| 序号 | 页面/组件 | 功能描述 | 依赖API |
|------|-----------|----------|---------|
| 1 | LoginPage | 用户登录 | POST /api/auth/login |

### 输出要求
- 页面/组件代码
- API 对接代码
- 本地测试通过
```

**输出格式：**
```json
{
  "task_id": "TASK_FE_001",
  "agent_type": "frontend",
  "status": "completed",
  "summary": {
    "total": 5,
    "completed": 5,

  },

    {
      "item": "LoginPage",
      "status": "completed",
      "files": ["src/pages/Login.vue"],
      "notes": "登录页面开发完成"
    }
  ]
}
```

### 3. 后端专家 (python-expert)

**职责：**
- API 接口设计与实现
- 数据模型设计
- 业务逻辑处理
- 数据库操作
- 单元测试编写

**输入格式：**

```markdown
- 数据模型定义
- 单元测试通过
```

### 4. 测试专家 (test-expert)

**职责：**
- 功能测试执行
- 集成测试执行
- Bug 记录与报告
- 测试报告生成

**输入格式：**
```markdown

| 功能点 | 测试类型 | 优先级 |
|--------|----------|--------|

### 输出要求
- 测试用例执行
- Bug 列表
- 测试报告
```

## 调度策略

### 任务分解原则

1. **按职责分组**
   - 页面/组件/交互 → 前端专家
   - API/数据/业务逻辑 → 后端专家
   - 测试/验证 → 测试专家

2. **按优先级排序**
   - P0 任务优先处理
   - 核心功能先行

3. **按依赖关系排序**
   - 无依赖任务并行执行
   - 有依赖任务串行执行

### 依赖关系处理

```
无依赖任务 (可并行)
├── 前端: 页面布局、静态组件
└── 后端: 数据模型、基础框架

有依赖任务 (需串行)
└── 后端 API 实现 ──▶ 前端 API 对接

最终依赖
└── 前后端完成 ──▶ 联调 ──▶ 测试
```

### 并行执行策略

**推荐策略：最大并行**

```
时间线:
后端 ████████████████████████
前端 ████████░░░░████████████
联调                 ████████
测试                         ████████
```

- 后端先启动，完成 API 设计
- 前端同步开发页面布局
- 后端 API 完成后，前端对接
- 联调验证
- 测试验收

## 结果汇总

### 汇总流程

1. 等待所有 Agent 完成
2. 收集各 Agent 结果
3. 合并统计数据
4. 生成进度报告

## 错误处理

### Agent 执行失败

| 失败类型 | 处理方式 |
|----------|----------|
| 超时 | 标记未完成，记录已完成部分 |
| 依赖阻塞 | 等待上游完成或标记阻塞 |
| 任务失败 | 记录失败原因，继续其他任务 |

### 重试机制

- 最大重试次数: 2
- 重试间隔: 60 秒
- 可重试场景: 超时、连接错误

---

## 最佳实践

1. **合理分配任务** - 根据复杂度均衡分配
2. **设置合理超时** - 前端/后端各 8 小时
3. **监控执行状态** - 实时查看进度
4. **结果及时汇总** - Agent 完成后立即收集
5. **迭代优化** - 分析效率，改进调度
