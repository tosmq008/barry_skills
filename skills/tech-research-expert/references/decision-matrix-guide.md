# 方案决策矩阵指南

## 核心理念

> 决策的质量取决于信息的充分性和评估的结构化程度。矩阵的价值不在于精确的数字，而在于**迫使决策者系统性地思考每个维度**。

## 加权评分矩阵法

### Step 1: 确定评估维度

**通用评估维度（根据项目需要选取并调整权重）：**

| 维度 | 说明 | 默认权重 | 调整场景 |
|------|------|---------|---------|
| **解决程度** | 方案对核心问题的解决程度 | 30% | 核心指标必须100%解决时可提高 |
| **实施代价** | 人力、时间、资金投入 | 20% | 资源极度紧张时权重提高 |
| **技术风险** | 技术可行性、稳定性风险 | 15% | 关键系统上线时权重提高 |
| **可维护性** | 长期运维和迭代的代价 | 15% | 长期运行系统权重提高 |
| **团队适配** | 团队技术能力匹配度 | 10% | 新团队或紧迫项目权重提高 |
| **扩展性** | 应对未来增长的能力 | 10% | 高增长业务权重提高 |

### Step 2: 评分标准定义

**标准化评分体系（1-5分）：**

| 分数 | 含义 | 说明 |
|------|------|------|
| 5 | 优秀 | 该维度表现突出，显著优于其他方案 |
| 4 | 良好 | 该维度表现良好，满足需求且有余量 |
| 3 | 达标 | 基本满足需求，没有明显优劣 |
| 2 | 不足 | 该维度存在明显短板，需要额外措施弥补 |
| 1 | 差 | 该维度存在严重问题，可能是致命缺陷 |

### Step 3: 执行评分

```
评分矩阵表：
┌──────────────┬──────┬───────┬───────┬───────┬──────────────────────────┐
│  评估维度     │ 权重 │方案 A │方案 B │方案 C │ 评分依据（必填）          │
├──────────────┼──────┼───────┼───────┼───────┼──────────────────────────┤
│  解决程度     │ 30%  │       │       │       │                          │
│  实施代价     │ 20%  │       │       │       │                          │
│  技术风险     │ 15%  │       │       │       │                          │
│  可维护性     │ 15%  │       │       │       │                          │
│  团队适配     │ 10%  │       │       │       │                          │
│  扩展性       │ 10%  │       │       │       │                          │
├──────────────┼──────┼───────┼───────┼───────┼──────────────────────────┤
│  加权总分     │ 100% │       │       │       │                          │
└──────────────┴──────┴───────┴───────┴───────┴──────────────────────────┘

加权总分 = Σ(维度权重 × 维度评分)
```

> ⚠️ **每个评分必须附带评分依据**，不可只给数字不给理由。

### Step 4: 敏感性分析

**检验决策的鲁棒性：**

```
敏感性检查：
1. 调整权重 ±10%，结论是否改变？
   - 如果稍微调整权重就改变结论 → 决策不够鲁棒，需要更多信息
   - 如果大幅调整权重仍保持结论 → 决策较为鲁棒

2. 调整评分 ±1 分，结论是否改变？
   - 如果某个维度评分变 1 分就翻转结论 → 该维度是关键决策因素，需重点论证

3. 如果去掉一个维度，结论是否改变？
   - 如果某个维度去掉后结论翻转 → 该维度的评估需要额外验证
```

## 杠杆分析法

**在评分矩阵之外，补充杠杆视角的分析：**

```
杠杆分析四要素：

1. 最大杠杆目标
   这个方案最核心要解决的一个问题是什么？
   不要试图一次解决所有问题，找到影响最大的那一个。

2. 杠杆支点
   方案的切入点在哪里？
   哪个组件/环节的改动能带来最大的全局改善？

3. 杠杆倍数
   投入 1 份资源，预期产出几倍效果？
   理想的杠杆倍数 > 3x。

4. 安全边际
   如果方案失败，最坏情况是什么？
   能否承受？有没有回滚方案？
```

## 决策输出模板

```markdown
## 方案决策

### 推荐方案: [方案名称]
**推荐等级**: 强烈推荐 / 推荐 / 有条件推荐

**核心理由**:
1. [理由1 + 数据支撑]
2. [理由2 + 数据支撑]
3. [理由3 + 数据支撑]

**杠杆分析**:
- 最大杠杆目标: [描述]
- 杠杆倍数: [预估]
- 安全边际: [描述]

### 备选方案: [方案名称]
**备选理由**: [在什么条件下应该切换到这个方案]

### 排除方案: [方案名称]
**排除理由**: [明确的排除原因]

### 取舍声明
选择推荐方案意味着:
- ✅ 获得: [列出获得的关键收益]
- ❌ 放弃: [列出放弃的东西]
- ⚠️ 承担: [列出需要承担的风险/代价]

这些取舍是否可接受: [是/否/需要讨论]
```

## 决策常见陷阱

1. **锚定效应**：第一个看到的方案会不自觉地成为"锚点"。对策：多人独立评分后再讨论。
2. **沉没成本**：因为已经投入了某个方案而不愿放弃。对策：评估时假设"从零开始会选什么"。
3. **群体思维**：团队趋同于某个声音大的观点。对策：先独立评分再集中讨论。
4. **过度自信**：对推荐方案的成功概率估计过高。对策：做敏感性分析和最坏情况推演。
5. **分析瘫痪**：过度追求完美的分析而延误决策。对策：设定决策 deadline，"足够好"即可。
