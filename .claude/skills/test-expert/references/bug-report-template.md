# Bug 记录模板

## Bug 严重程度定义

| 级别 | 名称 | 定义 | 示例 |
|------|------|------|------|
| S1 | Critical | 系统崩溃、数据丢失、核心功能不可用 | 登录失败、支付异常、数据库错误 |
| S2 | Major | 主要功能异常、影响用户正常使用 | 表单提交失败、页面加载错误 |
| S3 | Minor | 次要功能异常、有替代方案 | 排序功能异常、筛选不准确 |
| S4 | Trivial | 界面问题、体验问题 | 文案错误、对齐问题、颜色偏差 |

## Bug 优先级定义

| 级别 | 名称 | 定义 | 修复时限 |
|------|------|------|----------|
| P0 | 紧急 | 阻塞测试或上线 | 立即修复 |
| P1 | 高 | 影响主流程 | 当天修复 |
| P2 | 中 | 影响次要功能 | 3天内修复 |
| P3 | 低 | 优化类问题 | 版本内修复 |

## Bug 截图规范

### 截图命名规则
```
BUG_[编号]_[描述].png
例如：BUG_001_login_error.png
```

### 截图要求
- 清晰展示问题现象
- 包含必要的上下文信息
- 标注问题位置（如需要）
- 隐藏敏感信息

### 截图存放位置
```
docs/test/screenshots/
├── BUG_001_login_error.png
├── BUG_002_form_submit_fail.png
└── ...
```
